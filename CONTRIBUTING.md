# Contributing to etsy-oracle

Thanks for your interest. etsy-oracle is a free, open-source Etsy research tool, and contributions are welcome.

## Ground rules

- **Official Etsy API only.** This project does not scrape Etsy HTML. Pull requests that add scraping will be declined; scraping breaks on every Etsy change and puts the project at legal risk.
- **Estimates stay honest.** Sales and revenue are estimates derived from public signals. Never present an estimate as an exact figure, in code, the CLI, or the UI.
- **No secrets in commits.** API keys and `.env` files stay out of the repo.

## How to contribute

1. Open an issue describing the bug or feature before a large change, so we can agree on the approach.
2. Fork the repo and create a branch from `main`.
3. Keep pull requests focused. One change, one PR.
4. Make sure it runs. A change is not done because it compiles; run it and show the real output.
5. Open the pull request with a clear description of what changed and why.

## Development

Setup instructions land here as the build progresses. For now, see [docs/PRODUCT-BRIEF.md](./docs/PRODUCT-BRIEF.md) for the architecture and scope.

## Code of conduct

Be decent. Harassment or hostility is not welcome. Disagree on the work, not the person.
